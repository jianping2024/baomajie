# 宝马街第一版范围决策

决定日期：2026年6月23日

状态：已确认

## 1. 第一版目标

第一版不建设完整交易平台，目标收缩为：

> 抓取现有宝马街网站的公开数据，建立新的NestJS数据层和Next.js首页，验证新版信息架构、视觉方向、SEO和Web/PWA基础架构。

这里的首页不再是普通分类信息墙，而是“葡萄牙生活问题入口”：先承接在葡萄牙生活的中国人的真实问题，再把流量导向内容、服务、广告和订阅。

首版交付顺序：

```text
公开页面抓取
  → 数据清洗和标准化
  → 写入PostgreSQL
  → 建立Meilisearch索引
  → Next.js新版首页
  → 分类列表和基础详情
  → 基础站内消息
```

## 2. 已确认决策

| 问题 | 决策 | 实施影响 |
|---|---|---|
| 在线支付 | 第一版不包含 | 不开发Stripe、订单、订阅、退款和支付Webhook |
| 站内消息 | 第一版包含 | 需要用户登录、会话、消息、未读状态和基础通知 |
| 个人服务商 | 允许 | 服务商模型支持个人和企业两种主体 |
| 联系方式 | 公开 | 详情页可以展示，但必须提供用户控制和反滥用措施 |
| 旧用户及密码 | 暂不迁移 | 抓取内容不能自动归属到真实旧账号 |
| 旧数据库/API | 暂不提供 | 第一版数据源是公开HTML页面 |
| 首版重点 | 问题入口型首页 | 服务撮合、报价、认证和商业化后置 |

## 3. 第一版包含

- NestJS、Prisma和PostgreSQL基础工程。
- Next.js公开网站和响应式首页。
- 公开页面抓取器。
- 抓取任务、错误记录和来源映射。
- 数据清洗、去重和分类映射。
- 问题入口型首页。
- 分类导航。
- 搜索入口。
- 分类信息列表。
- 基础详情页。
- 公开联系方式展示。
- 新用户注册和登录。
- 基础站内消息。
- PWA Manifest和可安装基础能力。
- SEO元数据、Sitemap和旧URL关联。
- 基础管理后台，用于检查抓取结果和下架内容。

## 4. 第一版不包含

- 在线支付。
- 服务商会员和订阅。
- 线索购买。
- 置顶付费。
- 报价和预约闭环。
- 完整服务商认证。
- 旧用户、旧密码、收藏和历史消息迁移。
- 从旧数据库持续同步。
- 多国同时上线。
- 原生App商店上架。

## 5. 公开数据抓取边界

只抓取无需登录即可访问的公开页面，不绕过：

- 登录限制。
- 验证码。
- 访问权限。
- 管理后台。
- 私有API。
- 技术访问控制。

每条导入记录保留：

```text
source_url
source_id
source_type = public_html
fetched_at
content_checksum
last_seen_at
parser_version
```

抓取器必须：

- 限制并发和请求频率。
- 支持断点继续。
- 使用稳定的站点专用解析器。
- 记录解析失败页面。
- 根据校验值避免重复写入。
- 对HTML执行白名单清洗。
- 不因单条错误终止整个任务。
- 支持全量抓取和后续增量更新。

## 6. 抓取内容的所有权问题

因为暂不迁移旧用户，抓取的信息不能直接绑定到新用户账号。统一使用系统导入主体，并显示：

```text
来源：宝马街旧站公开信息
原始页面：source_url
发布者：旧站用户，尚未认领
```

后续提供“认领信息”流程：

1. 用户登录新系统。
2. 提交原联系方式验证。
3. 平台验证手机号、邮箱或其他证据。
4. 审核通过后，把信息转移给新账号。

在认领功能上线前，抓取内容默认只允许管理员编辑或下架。

## 7. 公开联系方式规则

用户决定公开联系方式，但仍需降低批量采集和诈骗风险：

- 详情页明确标记“公开联系方式”。
- 新发布内容由用户主动选择展示电话、邮箱、微信或WhatsApp。
- 抓取的旧内容按原公开状态导入，不推断隐藏字段。
- 电话号码可以在用户点击“显示联系方式”后呈现。
- 搜索引擎结构化数据中不输出私人电话号码。
- 联系方式区域加入诈骗和付款风险提示。
- 提供快速举报和申请下架入口。
- 对异常高频查看联系方式的IP和账号限流。
- 认领后允许用户修改或隐藏联系方式。

“点击后显示”不是强安全措施，但能减少普通爬虫直接批量采集。

## 8. 个人服务商模型

`organizations`不能强制每个服务商都是公司。建议增加主体类型：

```text
provider_subject_type = individual | organization
```

个人服务商：

- 关联一个用户。
- 可设置个人展示名称。
- 可提交个人资质。
- 不要求企业税号，但特定专业类别仍必须提供执业资格。

企业服务商：

- 关联组织。
- 支持成员和管理员。
- 可提交企业注册及专业资质。

第一版首页可以展示现有公开服务信息，但完整入驻、资质和认证流程后置。

## 9. 基础站内消息范围

第一版消息只实现：

- 新用户注册和登录。
- 用户从一条分类信息发起会话。
- 一对一文本消息。
- 未读数量。
- 会话列表。
- 拉黑和举报。
- 基础邮件或站内提醒。

暂不实现：

- 群聊。
- 语音和视频。
- 消息撤回。
- 端到端加密。
- 实时在线状态。
- 复杂文件附件。

抓取信息尚未被发布者认领时，不允许直接向旧发布者发送站内消息，只展示其原公开联系方式。

## 10. 第一版验收标准

### 数据

- 能稳定抓取首页和选定分类。
- 重复导入不会产生重复记录。
- 每条数据可以追溯原始URL。
- 标题、分类、地区、时间、正文和图片达到约定解析准确率。
- 抓取失败有可查看的错误记录。

### 页面

- 新首页在手机和桌面端正常显示。
- 首页数据来自新PostgreSQL/API，不在页面渲染时实时抓旧站。
- 分类列表和详情页有稳定URL。
- 公开页面具备SEO元数据。
- 联系方式规则和风险提示生效。

### 消息

- 新注册用户可对新系统原生发布的信息发起会话。
- 抓取且未认领的信息不会创建无接收人的站内会话。
- 拉黑和举报规则生效。

### 性能

- 首页LCP目标低于2.5秒。
- 首页API P95目标低于500毫秒。
- 抓取任务不影响网站API响应。
